# 보안 정책 (Security Policy)

## 지원 버전

| 버전    | 지원 |
|---------|------|
| 0.1.x   | ✅   |

`0.1.x` 는 개발 초기(0.y) 단계로, 보안 수정은 최신 `main` 과 다음 릴리스에 반영된다.

## 취약점 신고

보안 취약점은 **공개 이슈로 올리지 말고** 아래 경로로 비공개 신고해 주세요.

- **권장**: GitHub Security Advisories 의 "Report a vulnerability"
  ([보고 페이지](https://github.com/jaeyeonling/saju/security/advisories/new) — 비공개)
- 또는 이메일: `jaeyeonling@gmail.com`

가능하면 다음을 포함해 주세요 — 영향받는 모듈/버전, 재현 절차, 예상 영향, (가능하다면) 수정 제안.

## 대응 절차

- **접수 확인**: 영업일 기준 3일 이내
- **초기 평가**: 7일 이내 심각도 분류 및 대응 계획 공유
- 수정 후 패치 릴리스와 함께, 신고자 동의 시 크레딧을 명시한 advisory 를 공개합니다.

## 참고

이 프로젝트는 네트워크·파일 I/O·인증 기능이 없는 순수 계산 라이브러리라 공격 표면이 작습니다.
그래도 입력 처리·의존성(supply chain) 관련 문제 제보는 환영합니다. 의존성 자동 점검은
Dependabot, 코드 스캔은 CodeQL 로 상시 운영합니다.
